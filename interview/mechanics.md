# Interview Mechanics - The 45-Minute Framework

> Most candidates fail because they go silent or code too early. Use this exact flow.

---

## 1) The 45-Minute Clock

| Minute | What You Do | What You Avoid |
|---|---|---|
| 0-3 | Clarify input, edge cases, constraints | Coding immediately |
| 3-8 | Explain brute force + bottleneck | Silent thinking |
| 8-30 | Code while narrating decisions | Silent coding |
| 30-35 | Test on example + edge case | "I think it works" |
| 35-40 | Discuss optimization or tradeoff | Over-optimizing too early |
| 40-45 | Summarize + ask questions | Rushing extra features |

---

## 2) The Brute -> Optimize Script

1. State brute-force approach.
2. Identify bottleneck.
3. Propose optimization.
4. Confirm new complexity.

Use this script for every problem. Interviewers reward the process.

---

## 3) Stuck Recovery Protocol

If stuck, do this in order:

1. Draw a small example.
2. Solve a simpler version.
3. Read constraints for hints.
4. Pattern match explicitly.
5. Narrate your thought process.
6. Ask for one hint.

Never be silent for more than 2 minutes.

---

## 4) Code Quality Checklist

1. Clear variable names.
2. Edge cases handled early.
3. No magic numbers.
4. Complexity stated clearly.
5. Dry run with tricky input.
