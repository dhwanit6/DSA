# Phase 1 - Core DSA Foundations (Weeks 4-12)

> This phase builds your interview core. Do not reorder the blocks.

---

## 1) What "Done" Looks Like

By the end of Phase 1 you should be able to:

1. Solve most Easy and many Medium problems without syntax struggle.
2. Recognize core patterns quickly (window, two pointers, stack, DFS/BFS).
3. Explain brute-force to optimized transition clearly.
4. Write clean code with edge-case-first checks.

---

## 2) Weekly Map (No Guessing)

| Week | Focus | Output |
|---|---|---|
| 4-5 | Arrays + Strings + Math basics | Prefix/window/two-pointer confidence |
| 6 | Binary Search variants | Boundary and "BS on answer" comfort |
| 7-8 | Recursion + Backtracking | Template muscle memory |
| 9-10 | Linked List + Stack + Queue | Pointer confidence + monotonic stack |
| 11-12 | Trees + BST | DFS/BFS clarity + common tree patterns |

---

## 3) Daily Study Loop (90-150 min)

Run this same loop every day:

1. Learn (25-30 min): one focused concept.
2. Apply (30-40 min): 1 Easy + 1 Medium.
3. Push (35-45 min): one new Medium with 30 min honest struggle.
4. Journal (10 min): pattern trigger + mistake + reusable trick.

Use the [Patterns Journal](../topics/patterns-journal-template.md) for step 4.

---

## 4) Week-by-Week Execution

### Weeks 4-5: Arrays, Strings, Math

1. Master prefix sum, difference array, fixed/variable sliding window.
2. Practice two pointers on sorted inputs.
3. Implement merge sort once from memory.
4. Solve core set: LC 53, 238, 42, 3, 76, 560.

### Week 6: Binary Search

1. Practice exact-search and boundary-search templates.
2. Internalize `lower_bound` and `upper_bound`.
3. Solve BS-on-answer examples: LC 875, 1011, 1482.

### Weeks 7-8: Recursion and Backtracking

1. Use one template: choose -> explore -> undo.
2. Solve: LC 78, 90, 46, 39, 40, 79.
3. Draw recursion trees for your first two problems every day.

### Weeks 9-10: Linked List + Stack + Queue

1. Implement reverse list (iterative + recursive).
2. Solve cycle detection and entry point using Floyd.
3. Practice monotonic stack problems daily.
4. Solve: LC 23, 739, 146.

### Weeks 11-12: Trees and BST

1. Implement recursive + iterative traversals.
2. Solve height, diameter, LCA, max path sum.
3. Solve BST validation + kth smallest.
4. Solve: LC 124, 297, 105, 230.

---

## 5) Pattern Triggers to Memorize

| Trigger in Problem | Pattern to Try First |
|---|---|
| "Longest/shortest subarray with condition" | Sliding window |
| "Pair/triplet in sorted array" | Two pointers |
| "First true / minimum valid value" | Binary search on answer |
| "Generate all possibilities" | Backtracking |
| "Nearest greater/smaller element" | Monotonic stack |
| "Tree path/aggregate from children" | Post-order DFS |

---

## 6) Phase 1 Gate (Pass Before Phase 2)

You can move to [Phase 2](./phase-2-intermediate.md) when you can:

1. Solve a new prefix/window Medium in under 25 minutes.
2. Recognize monotonic stack pattern within 5 minutes.
3. Write one backtracking solution from scratch.
4. Solve one post-order tree DFS problem cleanly.

If you miss 2 or more: repeat one focused week on weak blocks first.
