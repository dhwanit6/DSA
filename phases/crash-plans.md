# Crash-Course Study Plans (For Late Starters)

> If you are late, the plan must be strict. Pick one timeline and follow it without edits.

---

## 1) Choose Your Timeline

- 4 weeks: Interview/OA in 1 month.
- 8 weeks: Some DSA done, need pattern mastery.
- 16 weeks: Intermediate start with 4-5 hours/day available.

If you are unsure, pick 8 weeks.

---

## 2) Non-Negotiable Rules

1. Talk aloud while solving.
2. Journal every solved problem (pattern + trigger + mistake).
3. One mock interview per week from Week 2.
4. No more than 45 minutes on a single problem before reading a hint.
5. Review cheatsheets before every mock.
6. Reserve one short weekly block for DBMS/OS/CN revision from Week 2 onward.

---

## 3) 4-Week Sprint (3+ Hours/Day)

### Week 1: Arrays, Binary Search, Two Pointers

1. Prefix sums: LC 303, 238, 560.
2. Sliding window: LC 3, 76, 209.
3. Two pointers: LC 15, 11, 167.
4. Binary search: LC 704, 33, 153.
5. Binary search on answer: LC 875, 1011, 410.
6. Kadane + intervals: LC 53, 56, 57.
7. Weekly review and journal catch-up.

### Week 2: Trees, Graphs, Stack

1. Tree DFS: LC 104, 543, 236.
2. Tree BFS: LC 102, 199, 297.
3. BST: LC 98, 230, 235.
4. Graph BFS/DFS: LC 200, 695, 733.
5. Topo + DSU: LC 207, 210, 547.
6. Monotonic stack: LC 739, 84, 42.
7. 45-minute mock + 30-minute DBMS basics review.

### Week 3: DP Essentials + OS/CN Quick Theory

1. 1D DP: LC 70, 198, 300.
2. Knapsack: LC 416, 494.
3. Grid DP: LC 62, 64, 221.
4. String DP: LC 1143, 72.
5. Interval DP: LC 516 or 312.
6. Quick theory: process vs thread, ACID, TCP vs UDP.
7. Full mock interview.

### Week 4: Peak Week

1. OA simulation (90 min, 3 problems).
2. 3 mocks in the week.
3. Behavioral prep: 6 STAR stories.
4. Quick revision:
   - DBMS
   - OS
   - Networks
5. Light review 48 hours before interview.

---

## 4) 8-Week Plan (2-3 Hours/Day)

1. Weeks 1-2: Arrays/Strings + Binary Search.
2. Weeks 3-4: Recursion/Backtracking + Stacks.
3. Weeks 5-6: Trees + Graphs.
4. Week 7: DP + DBMS/OS/CN oral revision.
5. Week 8: Mixed practice + mocks + final review.

---

## 5) 16-Week Plan (3-5 Hours/Day)

1. Weeks 1-2: Phase 0 + Arrays/Strings.
2. Weeks 3-4: Binary Search + Two pointers.
3. Weeks 5-6: Recursion + Linked list + Stack.
4. Weeks 7-8: Trees + Heaps + Tries.
5. Weeks 9-10: Graphs + DSU + Dijkstra.
6. Weeks 11-12: DP (all core patterns).
7. Weeks 13-14: DBMS + OS + Networks + Concurrency.
8. Weeks 15-16: OA + mocks + final review.

---

## 6) Crash Fundamentals Priority

If you can only revise 5 theory blocks, use this order:

1. SQL joins, grouping, indexes
2. ACID + normalization
3. process vs thread + deadlock
4. TCP vs UDP + browser request flow
5. mutex vs semaphore + thread pool vs async

This is the highest-yield theory stack for late starters.
