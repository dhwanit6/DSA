# Phase 2 - Intermediate Mastery (Weeks 13-24)

> This phase converts you from "I know topics" to "I can handle real interview mediums under pressure."

---

## 1) Outcome You Must Reach

By the end of Phase 2 you should be able to:

1. Model graph problems correctly and pick BFS/DFS/Topo/Shortest Path fast.
2. Solve medium DP problems using a repeatable state-transition method.
3. Explain greedy vs DP decision with confidence.
4. Handle timed interview sets without panic.

---

## 2) Block-Wise Roadmap

| Week Block | Focus | Must-Have Output |
|---|---|---|
| 13-15 | Graphs | Solid BFS/DFS/Topo/DSU fundamentals |
| 16-18 | Greedy | Understand when greedy works and fails |
| 19-22 | Dynamic Programming | 1D/2D/Knapsack/Subsequence pattern control |
| 23-24 | Heaps + Tries + Bit Basics | Strong utility toolkit for interviews |

---

## 3) Weekly Execution Formula

Use this 6-day cycle every week:

1. Day 1: Concept learning + 2 guided problems.
2. Day 2: 2 Medium problems (same pattern family).
3. Day 3: 1 timed medium + review log.
4. Day 4: 2 Medium problems (new variation).
5. Day 5: 1 mixed problem without topic label.
6. Day 6: Revision of weakest 3 problems + pattern journal.

Day 7 is recovery + light review.

---

## 4) Block Details (Action Steps)

### Weeks 13-15: Graphs

1. Master graph representation and traversal setup.
2. Practice BFS and DFS for components, islands, and shortest unweighted paths.
3. Add topological sort (Kahn + DFS), cycle detection, DSU.
4. Finish with shortest path basics: Dijkstra and Bellman-Ford.

Priority problems: LC 200, 207, 210, 547, 127, 787.

### Weeks 16-18: Greedy

1. For each problem, write one sentence: "Why does local choice stay globally safe?"
2. Build counterexamples when greedy fails.
3. Compare with DP alternatives for similar-looking problems.

Priority problems: LC 55, 45, 134, 452, 621.

### Weeks 19-22: Dynamic Programming

Follow this order exactly:

1. 1D DP: LC 70, 198, 213.
2. Grid DP: LC 62, 64.
3. Knapsack family: LC 416, 494, 322, 518.
4. Subsequence DP: LC 1143, 300, 72.
5. Interval/String extension: LC 516, 312.

For every DP problem, write:

1. State definition.
2. Transition.
3. Base case.
4. Iteration order.
5. Space optimization option.

### Weeks 23-24: Heaps, Tries, Bit

1. Heaps: top-K pattern, two-heaps pattern, custom comparator comfort.
2. Tries: insert/search/prefix + one wildcard DFS problem.
3. Bit: set/check/toggle/count tricks + XOR identity.

Priority problems: LC 347, 295, 208, 211, 136, 287.

## 6) Phase 2 Gate (Before Phase 3 or Interview Sprint)

You pass when you can:

1. Pick correct graph algorithm for a new problem in under 5 minutes.
2. Solve one unseen DP medium with proper state and transition.
3. Explain one greedy success and one greedy failure example.
4. Complete a 45-minute mock with clear communication.

If your interviews are under 4 weeks away, skip Phase 3 and go to [Interview Mechanics](../interview/mechanics.md) + [Mock Sessions](../interview/mock-sessions.md).
