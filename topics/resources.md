# Resources - The Minimal Stack That Actually Works

> Too many resources is a hidden productivity bug. This page keeps your prep stack lean.

---

## 1) Core Resource Stack (Use Only These)

1. This guide for sequence and execution.
2. LeetCode for coding practice.
3. One video source (Striver or NeetCode) for stuck topics.
4. One tracker system (this app dashboard/planner or your own sheet).

---

## 2) How to Use Each Resource

1. Guide: decide what to study today.
2. LeetCode: execute practice blocks.
3. Video: unblock one concept, then return to coding.
4. Tracker: log completion and weak spots.

If a resource does not fit one of these four jobs, remove it.

---

## 3) Worked Weekly Schedule (2.5 Hours/Day)

1. Monday to Friday:
   - 35 min chapter study
   - 85 min problem solving
   - 20 min review/journal
2. Saturday:
   - 90 min mock or OA simulation
   - 45 min debrief + mistakes list
3. Sunday:
   - 60 min revision only
   - no heavy new learning

---

## 4) Common Failure Modes and Fixes

| Failure mode | Why it hurts | Fix |
|---|---|---|
| Collecting new channels/courses daily | Context switching kills momentum | Freeze resources for 4 weeks |
| Watching videos before attempting | Passive understanding feels like progress | 30-minute attempt first, then watch |
| No revision day | Patterns decay quickly | Lock one weekly revision block |
| Using 3 trackers | Metrics become inconsistent | Keep one source of truth |
